Visualizacin de valores faltantes en 2 variables

En la clase anterior vimos como visualizar datos de una variable para obtener parmetros estadsticos.

En esta clase vamos a abordar el tema de 2 variables cuando contienen valores faltantes.

Veamos que tenemos un scatter plot (diagrama de puntos). Los puntos son observaciones (temperatura y humedad). Ahora que pasa con los puntos rojos? significa que tienen mediciones para una variable pero no para la otra. En el caso del eje horizontal x(tenemos medicin de temperatura, pero no de humedad), en el caso del eje vertical y (tenemos humedad pero no temperatura).

Lo graficamos de esta manera porque as no interfieren con las mediciones reales que tienen valor. Adems podramos incluir informacin extra como boxplot para ver la forma que tiene la distribucin de cada variable y saber en donde estn concentrados.

Por qu es importante? Porque cuando realizamos un anlisis exploratorio de valores faltantes, queremos saber si estos siguen una relacin o una tendencia; por ejemplo una regresin lineal, as llenaremos estos campos utilizando este mismo modelo (es decir el mismo comportamiento de los datos), agrupndolos a su variabilidad.

Entonces este tipo de grfica nos va ayudar a entender donde se encuentran los valores faltantes en un caso real.

Graficamos y vemos los puntos azules en los cuales nosotros tenemos observaciones para ambas y despus los puntos naranjas para los cuales solo tenemos observacin de una variable. As podemos ver el comportamiento para 2 variables.

Visualizacin de valores faltantes para 2 variables

Lo que sucede es que una variable tena una medicin y la otra no, entonces cmo los podemos graficar si no tenemos la medicin de otra variable? Nosotros tenemos que agregar un valor dummy un valor por defecto que nos permita visualizar en ese momento.

Vamos a crear una funcin para rellenar con valores dummies

proportion_below es un porcentaje en el cual nosotros queremos graficar los datos para no mezclarlos con los valores reales. Por eso elegimos una proporcin ms bajo, debajo del rango de observacin de valores reales.

jitter son puntos variables y aleatorios que nos ayudan a visualizar nuestros valores en caso de solapamiento.

seed debido a que introducimos un componente aleatorio, tambin necesitamos este parmetro que llamamos semilla y lo definimos como como 42 porque es un nmero simblico.

Lo que nos regresa la funcin es una serie.

Queremos graficar por debajo de nuestro rango de variables para que no se traslapen.

column_shifted = column.min() - column.min()*proportion_below

Esta instruccin es para ajustar nuestros datos en una zona donde no interfiera con nuestros datos reales.

In[4]:
def column_fill_with_dummies(
        column:pd.Series, # o puede ser string
        proportion_below: float = 0.10,
        jitter: float = 0.75,
        seed: int=42
) -> pd.Series:
    #Que har nuestra funcin
    #copia para evitar modificar datos reales
    column = column.copy(deep=True)
    # Extraer valores
    missing_mask = column.isna()
    # Conteo de valores faltantes
    number_missing_values = missing_mask.sum()
    # Rango de nuestras variables
    column_range = column.max() - column.min()

    #Hacer shift a nuestros datos
    #Shift data
    column_shifted = column.min() - column.min()*proportion_below

    #Crear ruido para diferenciar el solapamiento
    #Add jitter around the points 
    np.random.seed(seed)

    #Columna con ruido
    column_jitter = (np.random.rand(number_missing_values)-2)*column_range*jitter

    #Guardar conjunto de datos 
    #Dummy data
    column[missing_mask] = column_shifted + column_jitter

    return column
